## Acceptance

- Flat and nested JOIN syntax compile with source parameters declared once.
- `select { [user.id, company.id] }` exposes the resolved generated properties on the outer JOIN result.
- `A JOIN (B JOIN C)` and `(A JOIN B) JOIN C` produce distinct AST shapes and expected SQL parentheses.
- Table, derived `KSelectable`, union query, and nested `JoinSource` can each be a JOIN operand.
- Same-table/self-join source identity does not collapse by table name.
- Before `select`, query modifiers/execution are unavailable; after `select`, relation-construction methods are unavailable.
- The selected JOIN query supports normal execution, page/cursor stages, derived `select`, scalar/predicate subqueries, insert-select/CTAS where valid, and union.
- No `/tmp` debug write or old statement-style forwarding overload remains.
