# H2 Dialect Compatibility

## Confirmed Behavior

- H2 uses `MERGE` for upsert. A conditional update belongs after `WHEN MATCHED AND`, and update targets must be unqualified columns; target-table predicates are rewritten to the merge target alias.
- H2 does not support DML `RETURNING` or `FOR SHARE`; the dialect validator rejects both forms while retaining `FOR UPDATE`.
- H2 reports canonical type names such as `DOUBLE PRECISION`, `NUMERIC`, and `TIMESTAMP`. Render `FLOAT` as `DOUBLE PRECISION`, normalize `DECIMAL`/`NUMERIC` to `NUMERIC`, and combine numeric or temporal precision metadata with `COALESCE(C.NUMERIC_SCALE, C.DATETIME_PRECISION)`. H2's default timestamp precision is 6, so treat that value as the unspecified temporal precision during comparison; repeated schema syncs then remain idempotent.

## Verification

- `DatabaseStatementsTest` covers H2 metadata normalization and canonical type comparisons.
- `H2DialectIntegrationTest` covers real precision metadata, a second no-op schema sync, and conditional `MERGE` execution against H2 2.4.240.
