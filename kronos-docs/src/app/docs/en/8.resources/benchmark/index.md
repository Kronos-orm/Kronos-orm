## Read the benchmark report

The benchmark page embeds the report generated by the [`orm-benchmark-project`](https://github.com/Kronos-orm/orm-benchmark-project). The report contains measurements for `Kronos`, `JPA`, `MyBatis`, `Jimmer`, and `Ktorm` using `kotlinx-benchmark` and `JMH`.

Treat the numbers as output from that benchmark project, not as a release-version guarantee or a claim about every production workload. Read them together with the API behavior documented in the query, mutation, and database pages.

For application decisions, compare the report with your own workload shape: query complexity, write volume, database dialect, connection pool, transaction boundary, and SQL logging settings all affect runtime behavior.
